# 分级阈值参考表 (pg-host-resource-risk)

统一分级符号：🔴 严重 / 🟠 警告 / 🟡 关注 / 🟢 正常

## 1. 磁盘剩余空间（对每个关注目录独立判定）

| 级别 | 条件 |
|------|------|
| 🔴 严重 | 剩余空间 < 5% **且** 绝对值 < 10GB |
| 🟠 警告 | 剩余空间 < 10% **或** 绝对值 < 50GB |
| 🟡 关注 | 剩余空间 < 20% **或** 绝对值 < 100GB |
| 🟢 正常 | 以上均不满足 |

关注目录清单：`$PGDATA`、独立挂载的 `pg_wal`、日志目录（`pg_log` 或 `/var/log/postgresql`）、`/tmp`、所有自定义表空间路径（来自 `pg_tablespace`）、根目录 `/`。

## 2. OOM / 内存

| 级别 | 条件 |
|------|------|
| 🔴 严重 | MemAvailable < 5% 总内存 **或** Swap 使用率 > 80% |
| 🟠 警告 | MemAvailable < 10% **或** Swap 使用率 > 50% |
| 🟡 关注 | MemAvailable < 20% |
| 🟢 正常 | 以上均不满足 |

PG 进程重点排查阈值：RSS > 500MB **或** %MEM > 10%。

内存消耗构成估算公式：
```
理论最大占用 ≈ shared_buffers
             + max_connections × (5~10MB 基础开销)
             + 并发活跃查询数 × work_mem（按最坏情况：每查询可能用多份 work_mem，如多路排序/并行）
             + autovacuum_max_workers × maintenance_work_mem
```
若该值明显超过物理内存，即为潜在超卖风险，即使当前 MemAvailable 尚正常也应在报告中提示。

## 3. IO 使用率与等待

| 级别 | 条件 |
|------|------|
| 🔴 严重 | %util > 95% **且** await > 50ms |
| 🟠 警告 | %util > 80% **或** await > 20ms（HDD）/ 5ms（SSD） |
| 🟡 关注 | %util > 60% |
| 🟢 正常 | 以上均不满足 |

磁盘类型判断：`cat /sys/block/<dev>/queue/rotational`，`0` = SSD，`1` = HDD（旋转盘）。

检查点健康判断：若 `checkpoints_req` 远大于 `checkpoints_timed`（如 > 2倍），提示 `max_wal_size` 可能偏小。

## 4. 网络带宽利用率

| 级别 | 条件 |
|------|------|
| 🔴 严重 | 利用率 > 90% |
| 🟠 警告 | 利用率 > 70% |
| 🟡 关注 | 利用率 > 50% |
| 🟢 正常 | 以上均不满足 |

利用率 = (rx速率 + tx速率) / 网卡额定带宽。额定带宽来源：`ethtool <iface>` 或 `/sys/class/net/<iface>/speed`（单位 Mbps，需换算为与流量速率一致的单位）。

## 使用提示

- 所有阈值为通用经验值，不同硬件/业务场景可能需要调整，报告中应注明"以下为通用经验阈值，建议结合历史基线校准"。
- 非业务高峰期采集的数据，必须在报告中提示"当前非高峰，实际峰值可能更高"。
- 工具缺失时（`iostat`/`sar`/`iftop` 未安装）该维度只能给出降级结论，不可编造数据。
