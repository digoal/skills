# 报告输出模板 (pg-host-resource-risk)

按以下结构组织最终中文报告，占位符用实际探测结果替换：

```
📋 服务器资源风险评估报告

服务器：[hostname/ip]  OS：[版本]  评估时间：[now]
CPU：X 核  内存：Y GB  磁盘：Z GB（关注目录合计）  网络带宽：A Gbps
执行模式：[本地 / 远程(SSH)]
⚠️ [若非业务高峰期] 当前非高峰，实际峰值可能更高

🔴 严重风险：X 项  🟠 警告：Y 项  🟡 关注：Z 项  🟢 正常：N 项

━━━━━━━━━━━━━━━━━━━━━━
🔴 严重风险（立即处理）

【风险1】<维度：磁盘/内存/IO/网络> - <一句话风险名称>
  当前值：<实际值> | 阈值：<对应阈值>
  根因定位：<精确到目录/进程PID/SQL queryid/复制槽名/application_name>
```

## 撰写要点

- 每个风险条目必须给出「根因定位」而非仅罗列现象——例如不要只说"内存不足"，而要说"PID 12345（application_name=report_service）执行的 queryid=xxx 涉及大排序，temp_blks_written 异常高，占用 work_mem 可能达到 xGB"。
- 若某维度因工具缺失无法采集（如无 sysstat），在对应分组下用一行注明"⚪ 未检测：<原因>，建议安装 <包名>"，不要跳过不提。
- 修复建议要可执行、可复制粘贴（具体命令或具体参数值），避免空泛的"建议优化"。
- 全文使用中文输出。
