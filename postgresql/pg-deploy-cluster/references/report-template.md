# report-template.md — 部署报告结构说明

`scripts/09_generate_report.py` 会按以下结构渲染最终报告，本文件说明每个章节应包含的信息来源，
便于人工核对生成结果是否完整、真实、无编造内容。

1. **集群架构概述** — 来自 Step 1 用户确认的部署计划 + Step 6/7 实际搭建结果，包含节点角色表与
   Mermaid 拓扑图。
2. **连接信息** — 来自各节点的实际 IP/端口 + VIP/HAProxy 配置结果；密码字段必须调用
   `mask_password()` 脱敏后才能写入报告，任何情况下不得在报告正文出现完整明文密码。
3. **安装软件版本清单** — 来自 Step 3 安装脚本执行后 `psql --version` / `patronictl --version` /
   `repmgr --version` 等命令的真实输出，不得凭经验估计版本号。
4. **关键配置摘要** — 仅列出 Step 5 中生成的、偏离 PostgreSQL 默认值的参数，不需要列出全部参数。
5. **切换测试结果** — 来自 Step 9 `07_test_switchover.sh` / `08_test_failover.sh` 的真实执行记录
   （开始/结束时间戳、命令输出摘要、验证结论），若测试未执行或失败，必须如实注明，不得虚构"测试通过"。
6. **已知局限与建议** — 结合实际架构（是否有 witness、是否异步复制、DCS 节点数）给出针对性风险提示，
   不是泛泛而谈的通用清单。
7. **日常运维指引** — 给出与本次实际使用的 HA 软件（Patroni 或 repmgr）相匹配的具体命令，
   如日常 switchover 命令、从库重建命令、版本升级注意事项。

## 生成前检查清单

在调用 `09_generate_report.py` 前，确认以下事实已收集齐全并写入 `facts.json`：

- [ ] 所有节点的名称、角色、IP、端口
- [ ] 实际安装的软件版本（通过命令查询得到，而非用户输入的期望版本）
- [ ] switchover 测试的真实结果（时间、结论）
- [ ] failover 测试的真实结果（时间、结论），若未执行需注明原因
- [ ] 密码字段来自当次部署使用的真实密码（用于脱敏展示），而不是占位符
- [ ] 已识别的风险项（如单点 DCS、无 witness、异步复制）
