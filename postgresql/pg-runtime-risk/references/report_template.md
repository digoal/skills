## 📊 风险雷达图（文字版）

用 10 分制为每个维度打分（10 = 高风险，0 = 无风险），按以下十个维度输出：

```
事务回卷：   [████████░░] X/10
序列回卷：   [██░░░░░░░░] X/10
冻结风暴：   [██████░░░░] X/10
复制延迟：   [████░░░░░░] X/10
逻辑槽异常： [██████████] X/10
WAL 归档：   [██░░░░░░░░] X/10
WAL 堆积：   [████████░░] X/10
连接数耗尽： [██████░░░░] X/10
集群单点：   [██████████] X/10
大对象泄漏： [██░░░░░░░░] X/10
```

打分依据：该维度触发了严重/警告/关注中的哪一级，越高等级、涉及范围越广，分值越高；
未采集到数据（如权限不足跳过）应标注"数据不足，无法评分"而非强行给分。

## 🧭 后续建议

- 建议配置的监控告警项及阈值（引用 thresholds.md 中对应数值）。
- 建议的定期巡检频率，例如：
  - 事务年龄、序列剩余值：建议每日检查
  - 复制延迟、逻辑槽状态：建议每日或实时监控
  - 大对象泄漏：建议每月检查
- 若用户已有 pg-stat-snapshot 等定时快照采集基础设施，建议将数据库年龄、
  序列剩余值、复制延迟等指标纳入定时采集，形成趋势曲线而非单次快照。
